  • Behaviour-Driven Development

    Approach

    Test-driven development (TDD) and behaviour-driven development (BDD) have very little to do with tests. Sure, you write tests when doing BDD, but then again you do so when doing conventional development as well. (If you are not, consider it!)

    The difference is not about whether you write tests, but rather when and how you write them, within the software development process.

    In BDD, you are simply writing the specifications for your software in the form of tests. You specify that my program should do X, without having written X at all. You write a test case to test for this, run it, and obviously it will fail, as you have yet to write X.

  • Long-lived Models in Single Page Apps

    In single page applications (SPAs), when the users navigates from one “page” to another, what the browser really does is stay to on the same page, and use Javascript to re-render most of the DOM. No further HTTP requests are made by the browser to obtain a new HTML page, instead Javascript (usually the SPA framework) loads more data in the form of JSON, and more Javascript (usually the templating engine or the SPA framework) re-renders this as new DOM, replacing the old DOM. More Javascript - the SPA framework’s routing component - will also update the URL, using hash fragments, or the history API.
